# Обработка статей системными промптами

## Обзор

Система автоматически обрабатывает все импортированные статьи через цепочку AI-промптов:

1. **Фильтрация** - определение страны и категории
2. **Самари** - создание краткого описания
3. **Переписывание заголовка** - улучшение заголовка
4. **Кластеризация** - группировка по темам
5. **Определение продолжений** - выявление статей, поддерживающих "горение" темы
6. **Связи** - создание связей между похожими статьями

## Архитектура

### Сервисы

- **ArticleProcessingService** - основная обработка статей промптами
- **ClusterService** - создание кластеров тем и определение продолжений
- **RelatedArticlesService** - поиск и связывание похожих статей
- **AIService** - универсальный сервис для работы с AI API (OpenAI/Anthropic)

### Системные промпты

Промпты хранятся в БД в таблице `Prompt` с ключами:
- `article_filter` - фильтрация по стране и категории
- `article_summary` - создание самари
- `article_rewrite_title` - переписывание заголовка

## Настройка

### 1. Настройка AI API

Добавьте в `.env`:
```env
AI_PROVIDER=openai  # или anthropic
OPENAI_API_KEY=your_key_here
# или
ANTHROPIC_API_KEY=your_key_here
AI_MODEL=gpt-4o-mini  # опционально
```

### 2. Инициализация системных промптов

```bash
npx tsx scripts/init-system-prompts.ts
```

Или промпты создадутся автоматически при первом использовании.

### 3. Миграция БД

```bash
npx prisma migrate dev
```

## Использование

### Автоматическая обработка

Статьи обрабатываются автоматически после импорта из источников.

### Ручная обработка

#### Обработка одной статьи:
```bash
curl -X POST http://localhost:3000/api/admin/articles/process \
  -H "Content-Type: application/json" \
  -d '{"articleId": "article_id_here"}'
```

#### Обработка всех необработанных статей:
```bash
curl -X POST http://localhost:3000/api/admin/articles/process \
  -H "Content-Type: application/json" \
  -d '{"limit": 50}'
```

## Структура данных

### Article (новые поля)

- `country` - код страны (UA, RU, US, etc.)
- `category` - категория (политика, экономика, etc.)
- `summary` - самари статьи
- `rewrittenTitle` - переписанный заголовок
- `clusterId` - ID кластера тем
- `isContinuation` - является ли продолжением темы
- `processingStatus` - статус обработки (pending, processing, completed, failed)
- `processedAt` - дата обработки

### Cluster

- `name` - название кластера (тема)
- `description` - описание
- `isActive` - активен ли кластер (горящая тема)

### ArticleRelation

- `fromId` - ID исходной статьи
- `toId` - ID связанной статьи
- `similarity` - степень похожести (0-1)
- `reason` - причина связи

## Улучшение промптов

Промпты можно редактировать через админку (`/adminko/prompts`). 

### Рекомендации по улучшению:

1. **Фильтрация** - добавьте примеры стран и категорий
2. **Самари** - укажите желаемый стиль и длину
3. **Заголовки** - добавьте примеры хороших заголовков
4. **Кластеризация** - уточните критерии группировки
5. **Связи** - определите порог похожести

## Мониторинг

Статус обработки можно отслеживать через поле `processingStatus` в таблице `Article`:
- `pending` - ожидает обработки
- `processing` - обрабатывается
- `completed` - обработана
- `failed` - ошибка обработки

## Производительность

- Обработка одной статьи: ~5-10 секунд (зависит от AI API)
- Рекомендуется обрабатывать пакетами по 10-50 статей
- Обработка запускается асинхронно после импорта

## Troubleshooting

### Промпты не работают

1. Проверьте, что промпты созданы в БД
2. Проверьте, что `isActive = true`
3. Проверьте формат переменных в промптах (`{{variable}}`)

### AI API ошибки

1. Проверьте API ключ в `.env`
2. Проверьте лимиты API (rate limits)
3. Проверьте баланс аккаунта

### Статьи не обрабатываются

1. Проверьте `processingStatus` в БД
2. Проверьте логи сервера
3. Запустите обработку вручную через API
